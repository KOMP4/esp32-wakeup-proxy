# ESP32 Power Stick: Smart Hardware Proxy


Проект для надежного удаленного включения домашнего сервера (Wake-on-LAN). Сочетает в себе HTTP-прокси на сервере (VPS) и физическое устройство на базе микроконтроллера ESP32-C3, подключенное напрямую к материнской плате целевого сервера.

## 💡 Какую проблему решает этот проект?
Стандартный протокол Wake-on-LAN (WoL) часто оказывается бессилен, если домашний сервер находится за NAT провайдера или подключен через VPN-туннели (например, AmneziaWG), которые поднимаются только после загрузки ОС. 


**Преимущества этого решения:**
1. **Умная маршрутизация:** Python-прокси проверяет заголовок `Accept`. Если запрос идет от браузера, пользователь видит красивую HTML-страницу загрузки (с использованием Server-Sent Events). Если запрос идет от API/мобильного приложения, прокси моментально отдает `HTTP 503 Service Unavailable`, заставляя приложение корректно уйти в режим ожидания и повторить попытку позже без зависаний.
2. **Аппаратная независимость:** Команда на включение передается по проводу замыкания `PWR_SW` на материнской плату.
3. **Безопасность:** Передача команды осуществляется по протоколу MQTT с использованием двустороннего TLS-шифрования (mTLS). Подключиться к брокеру без клиентских сертификатов невозможно.

## ⚙️ Архитектура

`Браузер/Приложение` ➔ `Python aiohttp Proxy (VPS)` ➔ `MQTT Broker` ➔ `ESP32-C3 (за домашним роутером)` ➔ `Обвязка в виде оптопары` ➔ `Материнская плата (PWR_SW)`

>* **Необходимость в аренде VPS:** Для данного проекта необходим арендованный VPS или какой-то сервер с доступными портами, который будет включен постоянно. На нем будет работать сам прокси, так же крайне рекомендуется чтобы на этом устройстве стоял какой-либо self-host VPN и настроено подключение к целевому серверу исключительно через его подсеть, иначе целевой сервер будет постоянно включаться от сетевых ботов которые пингуют все доступные IP

### Ключевые особенности
* **NAT Keep-Alive:** Настроен частый пинг MQTT брокера (15 секунд), чтобы домашний роутер не закрывал простаивающий TCP-туннель.
* **Асинхронность:** Серверная часть написана на `aiohttp` и `aiomqtt`, что обеспечивает нулевое блокирование при проксировании тяжелого трафика

## 🛠 Аппаратная часть (Hardware)

### Bill of Materials (BOM)
* **Микроконтроллер:** ESP32-C3 SuperMini (или любой другой ESP32).
* **Оптопара:** PC817C (для гальванической развязки).
* **Резистор:** 220 Ом.
* **Светодиод:** Опционально, больше полезно для дебага.

### Схема подключения
| Пины ESP32 | Компонент | Пины материнской платы |
| :--- | :--- | :--- |
| `GPIO 4` | ➔ Резистор 220 Ом ➔ Анод (1) PC817C и '**+**' светодиода | |
| `GND` | ➔ Катод (2) PC817C и '**-**' светодиода |
| | Эмиттер (3) PC817C ➔ | `PWR_SW` (Минус / Ground) |
| | Коллектор (4) PC817C ➔ | `PWR_SW` (Плюс / Power) |


> **Настройки BIOS:** Для питания от USB-порта выключенного компьютера, убедитесь, что в BIOS включена функция подачи питания на USB в режиме ожидания (обычно требует отключения режима `ErP Ready` и работает только с `USB 3.0`).

## 🚀 Развертывание (Deployment)

### 1. Подготовка сертификатов (mTLS)
Сгенерируйте корневой сертификат (CA), сертификат сервера для Mosquitto и сертификат клиента для ESP32 и Python-скрипта. Положите их в папку `deploy/certs/`.

### 2. Запуск MQTT брокера
Перейдите в папку `deploy` и запустите брокер через Docker Compose. Он автоматически подхватит конфигурацию `mosquitto.conf`, оптимизированную для IoT.
```bash
cd deploy
docker compose up -d
```
### 3. Настройка и запуск Proxy-сервера

Настройте виртуальное окружение и установите зависимости:
```bash
cd server
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```
Скопируйте конфигурацию и укажите ваши данные:
```bash
cp config.example.json config.json
nano config.json
```
Для запуска в фоновом режиме используйте предоставленный шаблон Systemd-службы (deploy/smart-proxy.service).

### 4. Прошивка ESP32

Проект предпологает сборку через PlatformIO. Однако ничего не мешает прошить плату через Arduino IDE

    Откройте папку hardware/ в VS Code.

    Откройте main.cpp и вставьте ваши клиентские сертификаты и ключи Wi-Fi в соответствующие переменные.

    Скомпилируйте и прошейте устройство (в platformio.ini уже прописаны необходимые зависимости).
